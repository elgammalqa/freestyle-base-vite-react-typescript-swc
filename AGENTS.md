# AGENTS.md

## Project Context

This app was generated by [Buildra](https://buildra.dev). It's a **standard Vite +
React + TypeScript (SWC) + Tailwind CSS** project — no proprietary runtime, no
framework lock-in. You own this code: run it, edit it, deploy it anywhere.

Treat everything here as user-owned application code. Keep changes focused on the
user's request and preserve the existing project conventions.

## Setup & Run

```bash
npm install     # install dependencies
npm run dev     # start the Vite dev server (http://localhost:5173)
```

Other scripts:

```bash
npm run build   # production build → dist/
npm run preview # serve the production build locally
npm run lint    # eslint
```

## Key Files

- `src/` — application source. `src/App.tsx` is the entry component; `src/main.tsx`
  mounts it; `src/index.css` holds Tailwind directives and global styles.
- `src/components/ui/` — shadcn-style UI primitives. Reuse these before adding new
  component libraries.
- `index.html` — the single HTML entry; Vite injects the bundle here.
- `vite.config.ts` — Vite + plugin config, including the `@` → `src` path alias.
- `.buildra/skills/*.md` — the design and code conventions this project was built
  with. **Read these before making UI changes** — they cover styling, UI
  components, animations, icons, images, and React patterns. Follow them so new
  code matches what's already here.

## Working Notes

- **Match the existing conventions first.** Check `.buildra/skills/` and the
  surrounding code before introducing new dependencies, patterns, or styles.
- This is a **frontend-only** app (Vite SPA). There is no backend or database in
  this template; persistence is client-side unless the user adds a backend.
- The `@` alias maps to `src/` — import as `@/components/...`, not relative paths.
- Run `npm run lint` and `npm run build` before finishing a change to confirm it
  compiles and passes lint.
